AMENAZA ALTA

Microsoft Fixes 114 Windows Flaws in January 2026 Patch, One Actively Exploited

Resumen:

El Patch Tuesday de enero 2026 de Microsoft corrige 114 vulnerabilidades (8 críticas, 106 importantes). Se añadió a la lista KEV de CISA una falla de divulgación de información en Desktop Window Manager (CVE-2026-20805) que está siendo explotada activamente; otras correcciones notables incluyen un error de escalamiento de privilegios en VBS Enclave (CVE-2026-20876), problemas relacionados con certificados de Secure Boot y la remoción de controladores de módem Agere vulnerables.

Puntos clave:

  • Se corrigieron 114 vulnerabilidades: 8 críticas y 106 importantes.
  • CVE-2026-20805 (Desktop Window Manager) está siendo explotada y fue añadida a la lista KEV de CISA.
  • Otras correcciones importantes: VBS Enclave (CVE-2026-20876), problemas de Secure Boot y eliminación de drivers Agere vulnerables.

Detalles adicionales:

Priorizar la implementación de parches y mitigaciones relacionadas con las vulnerabilidades explotadas activamente y revisar sistemas que dependan de Secure Boot y controladores de módem antiguos.

AMENAZA MEDIA

[Webinar] Securing Agentic AI: From MCPs and Tool Access to Shadow API Key Sprawl

Resumen:

Los agentes de IA ahora construyen, prueban y despliegan código de forma autónoma, creando un riesgo oculto en el plano de control: los Machine Control Protocols (MCP). MCP inseguros y incidentes como CVE-2025-6514 (un proxy OAuth que derivó en RCE) demuestran que la automatización puede ser un vector de ataque. El webinar cubre los riesgos de MCP, claves API en la sombra, auditoría de acciones de agentes y controles prácticos para asegurar la IA agentiva.

Puntos clave:

  • Los MCP representan un plano de control que puede ser explotado por agentes autónomos.
  • Incidentes previos (ej. CVE-2025-6514) muestran cómo proxies o MCP inseguros pueden derivar en RCE.
  • El webinar ofrece controles prácticos: gestión de claves API, auditoría de acciones y restricciones de permisos.

Detalles adicionales:

Evaluar y auditar cualquier integración de agentes de IA, rotar y monitorizar claves API, y aplicar políticas de mínimo privilegio para herramientas automatizadas.

AMENAZA MEDIA

Model Security Is the Wrong Frame – The Real Risk Is Workflow Security

Resumen:

Incidentes recientes muestran que el riesgo principal en IA es la seguridad del flujo de trabajo, no tanto los detalles internos del modelo. Extensiones maliciosas de Chrome robaron chats de ChatGPT/DeepSeek de aproximadamente 900,000 usuarios y las inyecciones de prompt en repositorios de código engañaron a un asistente de IA para ejecutar malware. Las organizaciones deben inventariar el uso de IA, aplicar acceso con mínimo privilegio, escanear salidas y aplicar middleware de protección o monitoreo en tiempo de ejecución.

Puntos clave:

  • La seguridad del flujo de trabajo (workflow) es el principal vector de riesgo en despliegues de IA.
  • Casos reales: extensiones maliciosas exfiltraron chats (~900k usuarios) e inyecciones de prompt llevaron a ejecución de malware.
  • Recomendaciones: inventario de usos de IA, acceso con mínimo privilegio, escaneo de salidas y guardrails en middleware/tiempo de ejecución.

Detalles adicionales:

Implementar controles en cada punto del flujo de trabajo de IA y monitoreo continuo para detectar comportamientos anómalos o exfiltración de datos.

AMENAZA ALTA

Microsoft and Adobe Patch Tuesday, January 2026 Security Update Review

Resumen:

El Patch Tuesday de enero 2026 corrige 115 vulnerabilidades de Microsoft —incluyendo ocho críticas y tres zero-days (uno explotado activamente)—abordando RCE, escalamiento de privilegios y múltiples componentes de Windows; Adobe publicó 11 avisos corrigiendo 25 fallas (17 críticas). Las organizaciones deben priorizar parches y mitigaciones de inmediato.

Puntos clave:

  • Microsoft: 115 correcciones, 8 críticas, 3 zero-days (uno explotado activamente).
  • Adobe: 11 avisos que corrigen 25 vulnerabilidades, 17 de ellas críticas.
  • Se recomienda priorizar parches y aplicar mitigaciones mientras se despliegan actualizaciones.

Detalles adicionales:

Coordinar ventanas de mantenimiento, validar mitigaciones compensatorias y monitorear indicadores de compromiso relacionados con las vulnerabilidades conocidas.
